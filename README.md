# Chat-app | Онлайн-чат на Express.js, WebSocket, GraphQL, Apollo Client/Server, PostgreSQL

## Описание проекта

Пример реализации онлайн-чата, в котором есть авторизация, чат с разделением на комнаты, список пользователей, отображение статусов пользователей (онлайн, оффлайн). Переписка обновляется с помощью вебсокетов и хранится в базе данных. Apollo Server для GraphQL API и Apollo Client на фронтенде. REST для авторизации/регистрации, GraphQL для остальных запросов и WebSocket для доставки сообщений.
Для работы с БД используются модели: Chat, User, Message.

GraphQL используется для:

- Получения списка пользователей
- Получения чатов и сообщений
- Отправки сообщений (как альтернатива REST).

# Шаги по установке и запуску проекта:

1. Установить зависимости через **npm install**
2. **node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"** - генерация секретного ключа для работы с БД, добавить в энвик (SESSION_SECRET).
3. С официального сайта PostgresQL скачать программу: https://www.postgresql.org/download/. Проверка версии pgAdmin: psql --version.
4. Настройки установки бд: порт из энвика (DATABASE_URL), имя и пароль пользователя по умолчанию - postgres. Проверка подключения: psql -U postgres -d chat_app -h localhost -p 5432
5. Создать базу данных с именем chat_app (PostgresQL 18 - создать - База данных)
6. Инициализация таблиц в БД (chat_app): **npm run init-db**. Таблицы будут созданы в chat_app/Schemas/public/Tables.
7. Запустить сервер одним из способов:

- **npm run dev** - запуск через nodemon с автоматическим перезапуском при изменении кода в development mode
- **npm run start** - запуск через node без автоматического перезапуска в prodaction mode.

8. Дополнительно были установлены Apollo client/cerver, graphQL-зависимости через npm install @apollo/server @apollo/client graphql graphql-ws @graphql-tools/schema graphql-tag. Проверка в браузере: http://localhost:3000/graphql.

Приложение будет запущено на 3000 порту. Клиентский сервер отдельно запускать не требуется (фронт работает через статику в public/).

Посмотреть записи таблиц в БД: Нажми правой кнопкой на таблицу (например, users) → View/Edit Data → All Rows.

# Про роль GraphQL в проекте

GraphQL отлично подходит для работы со сложными данными. GraphQL в связке с WebSocket — это язык описания данных и способ взаимодействия. Он работает поверх WebSocket (или HTTP). Его задача — сделать общение по этому каналу структурированным, типобезопасным и удобным.

В контексте чата ключевая фишка GraphQL — Subscriptions (Подписки). Это не замена WebSocket, а надстройка над ним. GraphQL берет на себя логику "кто, на что подписался и какие поля хочет получать", а WebSockets доставляют эти данные в реальном времени.

- GraphQL используется для получения списка пользователей, получения чатов и сообщений, оправки сообщений. Для регистрации/авторизации используется REST.
- В проекте спроектированы типизированные graphql-схемы сообщения, пользователя, чата с типами (User, Message, Chat, Query, Mutation, Subscription) и резолверами (бизнес-логикой).
- В graphql-схеме определены основные точки входа (Query, Mutation, Subscriptions):
  1. резолверы для запросов (Query) получения списка сообщений и пользователей
  2. мутации (Mutation) для отправки новых сообщений и регистрации пользователей
  3. механизм подписок (Subscription) для мгновенной отправки сообщений, вывода статуса пользователей, показа последнего актуального сообщения в чате. Настроен как транспортный слой поверх WebSocket. Сообщения доставляются клиентам мгновенно.
- Бэкенд работает на Node.js/Express с Apollo Server. Apollo Server хранит данные в PostgreSQL и отправляет сообщения клиентам в реальном времени.
- Статус пользователя "онлайн" выставляется по активной сессии, которая берется из БД (в браузере сессия хранится в httpOnly куке).
